# Lollipop iOS build guide (full step-by-step)

This guide is for building the iOS client from scratch in Xcode with your own Apple team.

## A. Requirements

- macOS with Xcode 15+
- Apple Developer account (paid)
- NetworkExtension entitlement enabled for your team
- Real iPhone/iPad connected via cable (recommended)

## B. Create project from zero

1. Open Xcode -> **Create a new Xcode Project**.
2. Template: **iOS App**.
3. Product Name: `Lollipop`.
4. Team: select your Apple team.
5. Organization Identifier: e.g. `com.yourname`.
6. Interface: **SwiftUI**.
7. Language: **Swift**.
8. Use Core Data: unchecked.
9. Include Tests: optional.
10. Save project locally.

## C. Add Packet Tunnel extension target

1. In Xcode project navigator, select project root.
2. Click **+** under Targets.
3. Choose template: **Network Extension**.
4. Extension type: **Packet Tunnel**.
5. Product Name: `LollipopTunnel`.
6. Activate scheme when asked: **Yes**.

## D. Bundle IDs and signing

Set these in **Signing & Capabilities**:

- App target bundle id: `com.yourname.Lollipop`
- Extension bundle id: `com.yourname.Lollipop.LollipopTunnel`

Then update source code bundle id reference:

- In `LollipopVPNManager.swift`, set `providerBundleIdentifier` to your extension bundle id.

## E. Capabilities

For both app and extension targets:

1. Open **Signing & Capabilities**.
2. Add capability: **Network Extensions**.
3. For extension target choose packet tunnel support.
4. Ensure provisioning profiles are regenerated by Xcode.

## F. Replace generated files with repository files

Copy these files from repo into corresponding targets:

- App target:
  - `clients/ios/LollipopApp/LollipopApp.swift`
  - `clients/ios/LollipopApp/ContentView.swift`
  - `clients/ios/LollipopApp/LollipopVPNManager.swift`
- Extension target:
  - `clients/ios/LollipopTunnel/PacketTunnelProvider.swift`

When adding files, ensure target membership checkboxes are correct.

## G. What protocol switcher does

UI segmented switch:

- `wss` -> uses websocket transport branch in `PacketTunnelProvider`
- `https2` -> uses HTTPS polling branch (`/recv`) in `PacketTunnelProvider`

## H. Prepare certificate on device

1. Export server cert from docker: `cucumber.crt`.
2. Airdrop/email it to iPhone.
3. Install profile.
4. Enable trust in: **Settings -> General -> About -> Certificate Trust Settings**.

## I. Build and run on real device

1. Select app scheme `Lollipop` and your iPhone.
2. Build (Cmd+B).
3. Run (Cmd+R).
4. If asked to trust developer: open iPhone Settings -> VPN & Device Management and trust profile.

## J. First run flow

1. Open Lollipop app.
2. Fill server IP, port, path, username, password.
3. Choose protocol (`wss` or `https2`).
4. Tap **Save Profile**.
5. Tap **Connect**.
6. Check VPN icon/status on iOS.

## K. Troubleshooting

- If save/connect fails: verify `providerBundleIdentifier` matches extension bundle id exactly.
- If extension not starting: re-check NetworkExtension capability on both targets.
- If TLS errors: ensure cert is trusted system-wide on iOS.
- If no traffic: ensure server paths and credentials match nginx config.
